# Goal
Implement a fix for a GitHub issue based on a Root Cause Analysis (RCA) document.

# When To Use
Use this skill when you have an RCA document and need to implement the fix described therein.

# Instructions
1. Read the RCA document and relevant GitHub issue.
2. Verify the current state and reproduce the issue if possible.
3. Implement the fix as described in the RCA.
4. Add tests and run validation.
5. Verify the fix manually.

# Constraints
- Follow the specific fix strategy in the RCA.
- Ensure all validation passes.

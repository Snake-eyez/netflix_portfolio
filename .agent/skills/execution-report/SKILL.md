# Goal
Document the results of an implementation task, including modifications, validation results, and divergences.

# When To Use
Use this skill immediately after completing an implementation task.

# Instructions
1. Reflect on the implementation process.
2. Gather data on files changed and validation results.
3. Document any divergences from the original plan.
4. Save the report to the execution-reports directory.

# Constraints
- Be honest about challenges and divergences.
- Include specific validation outputs.

# Goal
Build a comprehensive understanding of the codebase structure, patterns, and context.

# When To Use
Use this skill at the beginning of a session or when switching to a new unknown codebase.

# Instructions
1. Analyze project structure and file list.
2. Read core documentation (README, CLAUDE.md).
3. Identify and read key files (entry points, config).
4. Generate a summary report of the codebase state.

# Constraints
- Do not modify code during priming.
- Focus on understanding high-level architecture.
